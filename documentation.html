Designing a responsive grid system.

By: Saad Aboussabr

23rd August 2022

I. Introduction:

I didn't start with building a responsive grid system. Matter of fact, I didn't even have responsivity in mind. I was thinking about a way to turn sections of the page into 3D objects.
hovering above an element with your mouse would make it 3D, by appplying a drop-shadow & lighter background-color to the element.

Luckily, When I got back home and started working on it.
It worked just as I intended

Hovering would make the element appear 3D by being lighter than the rest and having a minimal drop-shadow on the left + the added benefit of having a trail animation made the grid element rise and fall gracefully.

There was a problem though.

While reviewing and playing with the grid system: tuning the colors and drop shadow.
I tried moving the browser to another monitor for better inspection.
That's when I realised My original javascript function had only one screen width in mind,
I never thought about the other screen types, and how they would react to my grid.

That's when I set out to write, what at first seemed to be a fairly simple function, my bane for a couple of hours,
and the reason I began to hate the "height" property ever since.

II. Plan

  1. Make the grid container calcultate the width and height of the window.
  2. Make the grid element's width and height equal to the size of the window, devided by the number of grid elements there is.
  3. Add enough grid element rows to fill the whole page
  4. Apply changes every time the page resizes
